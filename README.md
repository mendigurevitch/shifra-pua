# שפרה ופועה — מערכת ניהול

אפליקציית PWA בעברית לניהול הארגון: יולדות, מתנדבות, ארוחות בוקר, מתנות, ערכות שבת ובוקר ליולדות.

נוספת למסך הבית של הטלפון כמו אפליקציה רגילה — בלי חנות אפליקציות. כשמעלים עדכון, כל המשתמשות מקבלות אותו בפתיחה הבאה.

---

## הפעלה מקומית (בדיקה על המחשב)

אין צורך ב-Node. כל דפדפן מספיק, אבל צריך שרת קטן (הדפדפן חוסם `file://`):

```powershell
# מתוך תיקיית הפרויקט
powershell -ExecutionPolicy Bypass -File serve.ps1
# ואז לפתוח http://localhost:8777
```

## העלאה לאוויר (GitHub Pages)

```bash
git remote add origin https://github.com/<שם-המשתמש>/shifra-pua.git
git push -u origin main
```

ואז ב-GitHub: **Settings → Pages → Source: main / root**.

הכתובת תהיה `https://<שם-המשתמש>.github.io/shifra-pua/` — זה הקישור שנשלח לכל המשתמשות.

**התקנה בטלפון:** פותחים את הקישור בכרום/ספארי → תפריט → "הוסף למסך הבית".

> בכל העלאת גרסה כדאי להעלות את `CACHE_VERSION` ב-`sw.js` (למשל `shifra-v2`), כדי שכל הטלפונים ימשכו את הקוד החדש מיד.

## מצב נוכחי: מקומי

כרגע הנתונים נשמרים **על כל מכשיר בנפרד** (localStorage). זה מצוין לבדיקות והתנסות, אבל:

- אין שיתוף נתונים בין המנהלת למתנדבות
- ההרשאות הן ויזואליות בלבד (מסתירות מסכים), לא אכיפה אמיתית

## מעבר לסנכרון בין כולן (Supabase)

1. פותחים פרויקט חינמי ב-`https://supabase.com`
2. **SQL Editor → New query** → מדביקים את כל `supabase/schema.sql` → **Run**
3. **Authentication → Users → Add user** — יוצרים משתמשת למנהלת הראשית
4. מריצים את שתי השורות האחרונות ב-`schema.sql` עם ה-UUID שנוצר
5. **Settings → API** → מעתיקים `URL` ו-`anon key` אל `js/config.js`
6. דוחפים ל-GitHub

מרגע שמולאו הפרטים ב-`config.js`, האפליקציה **דורשת התחברות** עם מייל וסיסמה,
מושכת את כל הנתונים מהשרת בכל פתיחה, וכל כתיבה מסונכרנת מיד.
כל משתמשת מקבלת חשבון משלה דרך **Authentication → Users**, ורשומה תואמת
בטבלת `users` עם התפקיד שלה (`admin` / `manager` / `volunteer`).

> למתנדבת צריך גם למלא את `volunteer_id` ברשומת ה-`users` שלה — כך המערכת
> יודעת אילו שיבוצים שייכים לה.

מרגע זה ההרשאות נאכפות במסד עצמו (Row Level Security): מתנדבת **לא יכולה** למשוך נתונים שאינם שלה גם אם תנסה.

## הרשאות

| תפקיד | מה רואה |
|---|---|
| מנהלת ראשית | הכל, כולל ניהול משתמשות |
| מנהלת משנית | הכל חוץ מניהול משתמשות; יכולה לשלוח הערה שתופיע כבאנר אצל המנהלת הראשית |
| מתנדבת | רק השיבוצים שלה + מסך הזמנת ציוד |

החלפת משתמשת: כפתור המשתמש בפינה העליונה (במצב מקומי — מאפשר גם לראות איך המערכת נראית לכל תפקיד).

## אוטומציות

- **יולדת חדשה** → נפתחים אוטומטית 5 ימי ארוחות (מדלג על שבת) + רשומת מתנת לידה + מתנת גיל שנה בתאריך היעד
- **ערכת שבת** → מוצעת אוטומטית שבוע אחרי הארוחה האחרונה
- **מתנת גיל שנה** → תזכורת שבוע לפני יום ההולדת
- **מלאי** → התראה אוטומטית בהגעה לסף המינימום; הזמנת מתנדבת מורידה מהמלאי
- **ציר זמן** → כל אירוע (ארוחה, מתנה, ערכה, מפגש) נרשם אוטומטית בפרופיל היולדת

## הודעות וואטסאפ

המערכת **לא שולחת בעצמה** — היא מכינה את ההודעה עם כל הפרטים האישיים ופותחת את וואטסאפ עם הנמען וההודעה מוכנים. לחיצה אחת על "שלח" והיא יוצאת.

זה מכוון: אין עלות, אין צורך במספר עסקי ובאישור מ-Meta, ואפשר לתקן את ההודעה לפני שליחה. אם בעתיד תרצו שליחה אוטומטית לגמרי — צריך WhatsApp Business API (כרוך בעלות ובאישור).

מרכז התזכורות (איקון הפעמון) מרכז כל מה שדורש טיפול היום.

## מבנה

```
index.html            מעטפת
css/style.css         עיצוב (RTL, משתני צבע)
js/config.js          הגדרות + חיבור Supabase
js/db.js              שכבת נתונים + לוגיקה עסקית
js/ui.js              רכיבי ממשק (מודל, טפסים, טוסט)
js/messages.js        תבניות וואטסאפ
js/screens-core.js    דשבורד, יולדות, מתנדבות, אנשי קשר, מלאי
js/screens-flows.js   ארוחות, ערכות שבת, מתנות, מפגשים, דוחות
js/app.js             ראוטר, תזכורות, הגדרות
supabase/schema.sql   סכמה + הרשאות
```
